LWC with Playwright

End to End Test Automation with CodeceptJS - Playwright

⭐️ Complete Example of E2E Tests with Playwright is available on github: salesforce-lwc-e2e-playwright

☁︎ Automate Salesforce Lighting Web Components with Micorsoft’s Playwright Browsers

💯 ADVANTANGE: The Shadow Locators are equivalent to Document.querySelector. Makes it simpler to Automate WebComponents or LWC Apps.

💁‍♂️ About LWC

Lightning Web Components is open source, empowering you to explore the source code, customize the behavior for your needs, and build enterprise-ready web components on any platform, not just Salesforce.

🛑 Challenge with Selenium Webdriver

As per the LWC Testing Docs, LWC uses a shadow DOM polyfill. There are known challanges with Selenium WebDriver to locate the elements but Codeceptjs-BDD has solved it to the extent. If you’re interested in LWC with Selenium Webdriver, take a look at LWC with Selenium.

🎊 Let’s make is simpler with Codeceptjs-BDD

The LWC Shadow Locators are equivalent to Document.querySelector. Take a look at the Example below!

  • Simplified Locators: Standard CSS/Xpaths Locators
I.click({ css: '.checkout-button' });

  • Healdess: Runs Headless on all three majorly used browsers,

    • Chromium
    • WebKit Safari
    • Firefox
  • Non-Headless: User can choose to run tests in Non-Headless mode too,
  SHOW=true yarn acceptance
  • Parallel Executions: Runs all Scenarios or all Test Suites in Parallel
  yarn acceptance:parallel
